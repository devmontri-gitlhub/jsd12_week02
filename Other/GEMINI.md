# Gemini CLI Project Mandates

This file contains foundational instructions for Gemini CLI when working on the **JSD12 Week-02 E-Commerce & SQL Project**.

## Role & Objectives
- **Peer Programmer:** Act as a senior developer and mentor for this learning project.
- **Context Awareness:** The project involves an e-commerce platform (HW/SW/Service) and a set of PostgreSQL exercises.
- **Bilingual Support:** Support both English and Thai in documentation and comments, as the project currently uses both.

## Engineering Standards
- **SQL Best Practices:** Ensure all SQL queries are clean, optimized, and follow the existing exercise structure (`postgresql/exercise_X.sql`).
- **JavaScript Conventions:** Follow the established snake_case naming convention (e.g., `id_product`, `name_product`) and use template literals or clear console logging for output.
- **Documentation:** Maintain the Thai problem statements and solutions in `01_my-ecommerce.md` while providing English translations or technical explanations where helpful.

## Workflow Mandates
- **Surgical Updates:** When modifying SQL exercises or JavaScript files, perform targeted changes that preserve the educational intent of the exercise.
- **Validation:** Always verify SQL syntax against common PostgreSQL standards.
- **Testing:** When adding features to the e-commerce logic, suggest simple test cases to verify functionality.

## Security
- Never expose or commit any real credentials if they are added during the development of login/registration features.
